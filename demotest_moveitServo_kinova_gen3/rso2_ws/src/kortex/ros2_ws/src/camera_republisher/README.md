# Camera Republisher Package

A ROS2 Jazzy C++ package that subscribes to camera and depth topics and republishes them, designed to work with `depth_image_proc` components in a single container for efficient processing.

## Package Structure

```
camera_republisher/
├── CMakeLists.txt
├── package.xml
├── include/camera_republisher/
│   └── camera_republisher_component.hpp
├── src/
│   └── camera_republisher_component.cpp
├── launch/
│   └── camera_republisher.launch.py
├── config/
│   └── default_params.yaml
└── README.md
```

## Features

- **Component-based architecture**: Runs as a composable node for efficient intra-process communication
- **Synchronized topics**: Uses message_filters to synchronize camera+camera_info and depth+depth_info pairs
- **Single configuration file**: All 9 topics (4 inputs + 5 outputs) configured in one YAML file
- **Dynamic remapping**: Launch file parses YAML and generates remappings automatically
- **Single container deployment**: Launches with depth_image_proc components in one container
- **Consistent topic routing**: All components use standard ROS2 remapping (no mixed approaches)

## Topics

### Input Topics (4)
- Camera image (default: `/camera/image_raw`)
- Camera info (default: `/camera/camera_info`)
- Depth image (default: `/camera/depth/image_raw`)
- Depth info (default: `/camera/depth/camera_info`)

### Output Topics (5)
- Republished camera image (default: `/camera_out/image_raw`)
- Republished camera info (default: `/camera_out/camera_info`)
- Republished depth image (default: `/camera_out/depth/image_raw`)
- Republished depth info (default: `/camera_out/depth/camera_info`)
- Point cloud (default: `/camera_out/points`) - generated by depth_image_proc

## How It Works

The launch file uses a clever approach:

1. **Parse YAML at launch time**: Loads your parameter file and extracts topic names
2. **Generate remappings**: Creates remapping tuples for all components dynamically  
3. **Consistent routing**: All components use standard ROS2 remapping (no parameters for topics)
4. **Single source of truth**: All 9 topics configured in one place

```python
# Launch file automatically does this:
params_data = load_yaml_params(params_file_path)
input_camera = params_data['camera_republisher_node']['ros__parameters']['input_camera_topic']
# ... generates remappings for all components
```

## Components in Container

1. **CameraRepublisherComponent**: Subscribes and republishes camera/depth data
2. **RegisterNode**: Registers depth images to RGB camera frame
3. **PointCloudXyzrgbNode**: Generates colored point clouds

## Building

```bash
cd ~/ros2_ws/src
git clone <your-repo>/camera_republisher
cd ~/ros2_ws
colcon build --packages-select camera_republisher
source install/setup.bash
```

## Usage

### Basic Launch
```bash
ros2 launch camera_republisher camera_republisher.launch.py
```

### Launch with Custom Parameters
```bash
ros2 launch camera_republisher camera_republisher.launch.py params_file:=/path/to/your/params.yaml
```

### Example Custom Parameters File
```yaml
camera_republisher_node:
  ros__parameters:
    # Input topics (4)
    input_camera_topic: "/my_camera/image_raw"
    input_camera_info_topic: "/my_camera/camera_info"
    input_depth_topic: "/my_camera/depth/image_raw"
    input_depth_info_topic: "/my_camera/depth/camera_info"
    
    # Output topics (5)
    output_camera_topic: "/processed/image_raw"
    output_camera_info_topic: "/processed/camera_info"
    output_depth_topic: "/processed/depth/image_raw"
    output_depth_info_topic: "/processed/depth/camera_info"
    output_points_topic: "/processed/pointcloud"
```

## Dependencies

## Performance Benefits

- **Intra-process communication**: Components in the same container share memory
- **Zero-copy message passing**: Efficient data transfer between components
- **Synchronized processing**: Message filters ensure temporal alignment
- **Single process**: Reduced overhead compared to multiple nodes

## Troubleshooting

1. **No output topics**: Check that input topics are being published
2. **Synchronization issues**: Verify that camera and camera_info have matching timestamps
3. **Missing dependencies**: Ensure depth_image_proc is installed: `sudo apt install ros-jazzy-depth-image-proc`

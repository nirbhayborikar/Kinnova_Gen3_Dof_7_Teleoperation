-------------- action error ---------
The error you're seeing, specifically goal ID already exists, combined with the std::runtime_error: Failed to accept new goal, is a classic ROS 2 Action timing issue.
The Root Cause

Your HandPosePublisher sends commands at 30Hz (from the rate parameter). 
Every time your camera processes a frame, it publishes to /gripper_control.

In your ArmControllerNode, the _gripper_cb is triggered every time that message arrives. 
Because 30Hz is very fast for an Action Server, you are calling self.gripper_client.send_goal_async(goal) 
while the previous goal is likely still being processed or "accepted" by the Robotiq controller.
 When you send goals that fast without checking if the previous one finished, the Action Client can
  reuse/clash IDs or overwhelm the server, leading to the crash.

. Can you close the gripper with a Twist command?

No, you cannot use the PicknikTwistController to control the gripper.

A Twist message (geometry_msgs/Twist) strictly contains 6 numbers: Linear X, Y, Z and Angular X, Y, Z. 
It is pure 3D spatial math meant only for the 6 (or 7) main joints of the arm.

Twist Linear Control (Translation Mode): This mode allows the end effector to be translated in Cartesian space (forward/backward, sideways, up/down) while keeping the orientation of the gripper fixed.
Twist Angular Control (Rotation Mode): This mode is used to change the orientation (pitch, yaw, roll) of the gripper (end effector) without altering its position.
Combined/Other Modes: A "Twist combined" mapping exists, which brings both Twist linear and Angular maps together. In this mode, you can toggle or switch to rotate the gripper by using specific buttons (often by holding a trigger or button) to switch the joystick controls from translation to orientation.
Gamepad Controls: The robot typically starts in Twist Linear mode. Switching between translation and rotation is done using the gamepad’s View or Menu buttons to cycle through control maps
